Purpose: These two classes are the foundation of the ConfigStore. ConfigCache ensures React components using useSyncExternalStore only re-render when their subscribed path's value actually changes. PubSubEmitter is the subscription mechanism that connects cache updates to subscriber notifications.

<feature>
  <name>ConfigCache with Structural Sharing (CORE-01)</name>
  <files>packages/core/src/config-cache.ts, packages/core/src/config-cache.test.ts</files>
  <behavior>
    ConfigCache stores config in memory with stable object identity for unchanged paths:

    - update(envelope) stores the config and returns an array of changed dot-paths
    - When config has NOT changed, update returns empty array and all references are identical (Object.is)
    - When a nested path changes (e.g., features.darkMode), only that path and its ancestors get new references; sibling paths keep their reference
    - get("features.darkMode") resolves dot-notation against the stored config
    - get("") returns the entire config object
    - get() on a non-existent path returns undefined
    - getConfig() returns the raw config (same as get(""))
    - getEnvelope() returns the full envelope or null if no data
    - Structural sharing handles: primitives, nested objects, arrays (positional comparison), null values
    - Root path "" is always included in changedPaths when any config changes

    Test cases:
    - Test 1: First update returns all paths as changed
    - Test 2: Identical update returns empty changedPaths, Object.is(prev, next) is true for all paths
    - Test 3: Change one nested key -> only that key's path and ancestors in changedPaths, sibling subtree keeps reference
    - Test 4: get("deep.nested.path") traverses correctly
    - Test 5: get("nonexistent.path") returns undefined
    - Test 6: Array positional comparison works (same elements same order = stable identity)
    - Test 7: getEnvelope() returns null before any update
    - Test 8: update() with changed config returns root "" in changedPaths
  </behavior>
  <implementation>
    Implement a `structuralShare(prev, next)` function that recursively compares objects and returns `prev` reference when unchanged. The ConfigCache.update() method:
    1. If no previous config, store the new config and return all leaf paths + ancestors + root
    2. If previous config exists, run structuralShare(prev, next), compute diff paths by walking both trees and collecting paths where Object.is returns false
    3. Store the structurally-shared result as the new config

    Implement `getByPath(obj, path)` for dot-notation resolution. Empty string returns the whole object.

    Import ConfigEnvelope from ./types.js.
  </implementation>
</feature>

<feature>
  <name>PubSubEmitter with Path-Level Subscriptions (CORE-02)</name>
  <files>packages/core/src/pubsub.ts, packages/core/src/pubsub.test.ts</files>
  <behavior>
    PubSubEmitter provides granular key-path subscription registry with path-level notifications:

    - subscribe(path, callback) registers a callback for a specific dot-path
    - subscribe returns an unsubscribe function; calling it removes the callback
    - emit(changedPaths) calls all callbacks whose subscribed path is in the changed set
    - emit does NOT call callbacks whose path is NOT in the changed set
    - Subscribing to "" (root) means the callback fires on ANY change
    - Multiple subscribers to the same path each get called independently
    - Calling unsubscribe twice does not throw
    - clear() removes all subscriptions
    - size getter returns total number of active subscriptions

    Test cases:
    - Test 1: subscribe + emit matching path -> callback called
    - Test 2: subscribe + emit non-matching path -> callback NOT called
    - Test 3: Root subscriber ("") fires on any emit
    - Test 4: Unsubscribe removes callback, subsequent emit does not call it
    - Test 5: Double unsubscribe does not throw
    - Test 6: Multiple subscribers on same path all fire
    - Test 7: clear() removes all, subsequent emit calls nothing
    - Test 8: size reflects current subscription count
  </behavior>
  <implementation>
    Use a `Map<string, Set<() => void>>` as the internal registry. subscribe() adds to the Set for a given path key, returns a function that deletes from the Set (and cleans up the Map entry if the Set is empty). emit() iterates the Map and calls callbacks where the key is in the changedPaths Set.

    No external dependencies needed.
  </implementation>
</feature>
