<feature>
  <name>HttpClient with Cache-Control and Visibility Change (CORE-03)</name>
  <files>packages/core/src/http-client.ts, packages/core/src/http-client.test.ts</files>
  <behavior>
    HttpClient fetches config via HTTP, respects Cache-Control, re-fetches on page visibility change:

    - Constructs URL as `${apiUrl}/project/${projectSlug}/config/${environmentSlug}` (strip trailing slash from apiUrl)
    - Attaches headers: `Accept: application/json` and `X-SkyState-Client: @skystate/core/0.1.0`
    - If `clientHeader` option is provided, uses that instead of default `X-SkyState-Client` value
    - start() triggers the initial fetch
    - On successful fetch (200): calls onUpdate(envelope), parses Cache-Control max-age, schedules next fetch via setTimeout
    - On failed fetch: calls onError(error) -- does NOT retry by default (per user decision: conservative, no thundering herd)
    - On visibility change to "visible": re-fetches immediately if cache is expired (Date.now() - fetchedAt >= maxAge * 1000), otherwise no-op
    - Does NOT fetch when page is hidden
    - Guards all browser APIs with `typeof document !== 'undefined'` for SSR/Node safety
    - dispose() clears the re-fetch setTimeout, removes the visibilitychange listener, aborts any in-flight fetch via AbortController
    - Sets a `disposed` flag; any async callback checks it before proceeding

    Test cases (use vi.fn() for fetch mock, vi.useFakeTimers for scheduling):
    - Test 1: start() fetches from correct URL with correct headers
    - Test 2: Custom clientHeader overrides X-SkyState-Client value
    - Test 3: Successful fetch calls onUpdate with parsed response body
    - Test 4: Failed fetch (network error) calls onError
    - Test 5: Failed fetch (404 status) calls onError with SkyStateError
    - Test 6: Cache-Control max-age=60 schedules re-fetch after 60 seconds
    - Test 7: No Cache-Control header -> no scheduled re-fetch (single shot)
    - Test 8: Visibility change triggers re-fetch when cache expired
    - Test 9: Visibility change does NOT re-fetch when cache still fresh
    - Test 10: dispose() cancels scheduled re-fetch, removes visibility listener
    - Test 11: Callback after dispose is ignored (disposed flag)
  </behavior>
  <implementation>
    HttpClient class with:
    - Private fields: apiUrl, projectSlug, environmentSlug, clientHeader, onUpdate, onError, fetchedAt, maxAgeMs, timerId, abortController, visibilityHandler, disposed
    - buildUrl(): constructs the fetch URL
    - buildHeaders(): constructs headers with X-SkyState-Client
    - start(): calls doFetch()
    - doFetch(): uses global fetch() with AbortController signal, parses response, calls onUpdate or onError, schedules next fetch
    - parseMaxAge(): regex extraction of max-age from Cache-Control header
    - scheduleRefetch(): setTimeout for maxAge * 1000 ms
    - onVisibilityChange(): checks freshness, conditionally refetches
    - setupVisibilityListener(): guards typeof document, adds event listener
    - dispose(): clearTimeout, removeEventListener, abort(), set disposed=true

    Import ConfigEnvelope from ./types.js, SkyStateError from ./error.js.
    Read package version from package.json or hardcode "0.1.0" as a constant.
  </implementation>
</feature>

<feature>
  <name>ConfigStore Facade and Singleton Registry (CORE-04)</name>
  <files>packages/core/src/config-store.ts, packages/core/src/config-store.test.ts, packages/core/src/index.ts</files>
  <behavior>
    ConfigStore composes ConfigCache + PubSubEmitter + HttpClient into the public API:

    - Constructor accepts ConfigStoreOptions
    - If initialConfig is provided, seeds ConfigCache immediately (wraps in a synthetic ConfigEnvelope with version 0.0.0 and current timestamp)
    - Sets isLoading=true and starts HttpClient
    - On HttpClient onUpdate: updates ConfigCache, sets isLoading=false, sets lastFetched=new Date(), clears error, emits changed paths via PubSubEmitter
    - On HttpClient onError: sets error, sets isLoading=false if this was the initial load, emits to notify subscribers of error state change
    - subscribe(path, callback): delegates to PubSubEmitter.subscribe(path, callback) -- returns unsubscribe function (useSyncExternalStore compatible)
    - getSnapshot(path): delegates to ConfigCache.get(path) -- returns stable reference (useSyncExternalStore compatible)
    - isLoading getter: returns current loading state
    - error getter: returns last error or null
    - lastFetched getter: returns last successful fetch timestamp or null
    - dispose(): calls HttpClient.dispose(), PubSubEmitter.clear(), removes from singleton registry

    Singleton registry (getOrCreateStore):
    - Module-level Map<string, ConfigStore> keyed by `${apiUrl}|${projectSlug}|${environmentSlug}`
    - getOrCreateStore(options) returns existing instance or creates new one
    - dispose() removes the store from the registry so a fresh instance can be created on next call

    initialConfig fallback chain (per user decision):
    1. Live fetched value (if available) -- returned by getSnapshot after successful fetch
    2. Key from initialConfig (if key exists) -- returned by getSnapshot before fetch completes
    3. In-code fallback value -- handled by React hook layer (Phase 3), not by core
    4. undefined -- returned by getSnapshot when no data at path

    Update packages/core/src/index.ts to export: ConfigStore, getOrCreateStore, ConfigCache, PubSubEmitter, HttpClient, SkyStateError, and all types.

    Test cases (use vi.fn() for fetch mock):
    - Test 1: Constructor with no initialConfig -> isLoading=true, getSnapshot returns undefined
    - Test 2: Constructor with initialConfig -> getSnapshot returns initialConfig values immediately, isLoading=true
    - Test 3: After successful fetch -> isLoading=false, getSnapshot returns fetched values, lastFetched is set
    - Test 4: After successful fetch with initialConfig -> fetched values replace initialConfig
    - Test 5: subscribe(path, cb) fires cb only when that path changes on update
    - Test 6: subscribe(path, cb) does NOT fire when unrelated path changes
    - Test 7: Re-fetch failure after initial load -> keeps cached config, surfaces error, isLoading stays false
    - Test 8: Initial fetch failure with no initialConfig -> error set, isLoading=false, getSnapshot returns undefined
    - Test 9: Initial fetch failure with initialConfig -> initialConfig preserved, error set, isLoading=false
    - Test 10: dispose() cleans up everything, removes from singleton registry
    - Test 11: getOrCreateStore returns same instance for same tuple
    - Test 12: getOrCreateStore returns different instance for different tuple
    - Test 13: After dispose + getOrCreateStore, a new instance is created
  </behavior>
  <implementation>
    ConfigStore class:
    - Private fields: cache (ConfigCache), pubsub (PubSubEmitter), http (HttpClient), _isLoading, _error, _lastFetched, _disposed, _registryKey
    - Constructor: create ConfigCache, PubSubEmitter, HttpClient; if initialConfig, seed cache; call http.start()
    - handleUpdate(envelope): cache.update(envelope) -> get changedPaths -> set _isLoading=false, _error=null, _lastFetched=new Date() -> pubsub.emit(changedPaths + all status paths)
    - handleError(error): set _error -> if first load set _isLoading=false -> pubsub.emit (notify for error state change)
    - subscribe/getSnapshot: delegate to pubsub/cache
    - dispose: http.dispose(), pubsub.clear(), remove from registry

    For status subscriptions (isLoading, error, lastFetched): use a special reserved path like "__status" that always fires on any state change. This allows React hooks in Phase 3 to subscribe to status changes.

    getOrCreateStore: module-level Map, key construction, store.dispose() calls registry.delete(key).

    Import ConfigCache from ./config-cache.js, PubSubEmitter from ./pubsub.js, HttpClient from ./http-client.js, types from ./types.js.
  </implementation>
</feature>
